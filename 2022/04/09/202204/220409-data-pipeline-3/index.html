220409 лҚ°мқҙн„° нҢҢмқҙн”„лқјмқёмқҳ кө¬м¶• - 3

мқҙлІҲ нҸ¬мҠӨнҢ…м—җм„ңлҠ” лҚ°мқҙн„° нҢҢмқҙн”„лқјмқёмқ„ кө¬м¶•н•Ём—җ мһҲм–ҙ м–ҙл–Ө л°©мӢқмңјлЎң м ‘к·јмқ„ н•ҙм•јн•ҳлҠ”м§Җ, к·ёлҰ¬кі  кі л Өн•ҙм•јлҗҳлҠ” л¶Җл¶„кіј мӢӨл¬ҙм—җм„ң мҳҲмҷёл°ңмғқмқҙ л°ңмғқн•ҳл©°, лҢҖмІҙл°©м•ҲмңјлЎң м–ҙл–Ө л¶Җл¶Җмқ„ м„ нғқн•ҙм•јлҗҳлҠ”м§Җм—җ лҢҖн•ҙ м •лҰ¬н•ҙліҙл Өкі  н•ңлӢӨ.

(1) Data & AI LANDSCAPE

2015л…„лҸ„ Bigdata landscapeм—җм„ңлҠ” м„ңл№„мҠӨк°Җ м…Җ мҲҳ мһҲмқ„ л§ҢнҒјмқҳ к°ҜмҲҳмҳҖлҠ”лҚ°, 2020л…„ кё°мӨҖмңјлЎң(мғҒлӢЁмқҳ мқҙлҜём§Җ м°ёмЎ°), м—¬лҹ¬ лҚ°мқҙн„° кё°л°ҳмқҳ нҡҢмӮ¬к°Җ л“ұмһҘн•ҳкі , лҢҖн‘ңм ҒмңјлЎң л„·н”ҢлҰӯмҠӨк°Җ л“ұмһҘн•ҳл©ҙм„ң л¶„м„қ л°Ҹ м¶”мІңм—җ л§ҺмқҖ кіөн—Ңмқ„ н•ҳмҳҖлӢӨ. мқҙлҹ¬н•ң м„ңл№„мҠӨл“ӨмқҖ мҳӨн”ҲмҶҢмҠӨнҷ”н•ҳлҠ” кІҪмҡ°к°Җ л§Һм•ҳкі , мҳӨн”ҲмҶҢмҠӨнҷ”лҗң м„ңл№„мҠӨл“ӨмқҖ нҒҙлқјмҡ°л“ң м§„мҳҒм—җм„ң мһҗмІҙ м„ңл№„мҠӨнҷ”н•ҳкё° мң„н•ҙ лҚ”мҡұ лӢӨм–‘н•ң м„ңл№„мҠӨл“Өмқҙ л“ұмһҘн•ҳкё° мӢңмһ‘н–ҲлӢӨ.

мөңк·јм—җ нҺҳмқҙмҠӨл¶Ғ, нҠёмң„н„°, лҰ¬н”„нҠёмқҳ кё°м—…л“Өмқҙ лҚ°мқҙн„° кё°л°ҳмңјлЎң л№„мҰҲлӢҲмҠӨлҘј нҷ•лҢҖн•ҳл©ҙм„ң, м—¬лҹ¬ м„ңл№„мҠӨл“Өмқ„ мҡҙмҳҒн•ҳкё° мӢңмһ‘н–ҲлӢӨ.

[м°ёкі ]
AirflowлҠ” Airbnbм—җм„ң л§Ңл“  м „мІҙ н”ҢлЎңмҡ°лҘј м»ЁнҠёлЎӨн•  мҲҳ мһҲлҠ” нҲҙ(workflow tool)мқёлҚ° мҳӨн”ҲмҶҢмҠӨнҷ”н•ҳл©ҙм„ң, мқҙ кІғмқ„ кё°л°ҳмңјлЎң нҒҙлқјмҡ°л“ң м§„мҳҒм—җм„ңлҠ” мһҗмІҙ м„ңл№„мҠӨнҷ”н•ҳкё° мң„н•ҙм„ң лӢӨм–‘н•ң м„ңл№„мҠӨл“Өмқҙ л“ұмһҘмқ„ н•ҳмҳҖлӢӨ.
к·ё мҷём—җлҸ„ лҚ°мқҙн„° мҳҲмёЎмқ„ мң„н•ҙ AI м§„мҳҒм—җлҸ„ лӢӨм–‘н•ң м„ңл№„мҠӨл“Өмқҙ л“ұмһҘн•ҳмҳҖлӢӨ.

(2) лҚ°мқҙн„° нҢҢмқҙн”„лқјмқё кө¬м¶•м—җ м ‘к·јн•ҳлҠ” л°©лІ•

мҡ°м„  л§ҺмқҖ ArchitectureмӨ‘м—җм„ң Lambda ArchitectureлЎң м ‘к·јн•ҳлҠ” мқҙмң лҠ”, лҚ°мқҙн„° нҢҢмқҙн”„лқјмқё кө¬м¶•н• л•Ң costмёЎл©ҙм—җм„ң м Ғкі , лӢӨм–‘н•ң ArchitectureлҘј кө¬м„ұн•  мҲҳ мһҲлӢӨлҠ” л©ҙм—җм„ң мһҘм җмқҙ мһҲкё° л•Ңл¬ёмқҙлӢӨ.

IoTкё°кё°лӮҳ SNS, мҠӨл§ҲнҠё кё°кё°лЎңл¶Җн„° лӘЁмқё лҚ°мқҙн„°лҠ” Raw Dataмқҳ нҳ•нғңлЎң Raw Data Storeм—җ м ҖмһҘлҗңлӢӨ. лҚ°мқҙн„°м—җ лҢҖн•ң л№„мҰҲлӢҲмҠӨм Ғ мҡ”мІӯм—җ мқҳн•ҙ Batch-Processing-Engineмқ„ нҶөн•ҙм„ң мІҳлҰ¬лҘј н•ҳкё°лҸ„ н•ҳкі , Real-Time-Processing-Engineмқ„ нҶөн•ҙм„ң мІҳлҰ¬лҘј н•ҳкё°лҸ„ н•ңлӢӨ.

л§Ңм•Ҫ Batch-Processingм—җ лҢҖн•ң л№„мҰҲлӢҲмҠӨм Ғ мҡ”кө¬к°Җ мһҲлӢӨл©ҙ Batch-Processing-Engineмқ„ нҶөн•ҙм„ң Serving Data Storeм—җ Raw Data Storeмқҳ лҚ°мқҙн„°лҘј м ҖмһҘн•ҳкІҢ лҗңлӢӨ. (ліҙнҶө мқҙ мқјл Ёмқҳ кіјм •мқ„ ETL(Extract/Transformation/Load)мһ‘м—…мқҙлқјкі  н•ңлӢӨ)

л§Ңм•Ҫ Real-Timeм—җ л№„мҰҲлӢҲмҠӨм Ғ мҡ”кө¬к°Җ мһҲлӢӨл©ҙ Real-Time Processing Engineмқ„ нҷңмҡ©н•ҙм„ң Serving Data Storeм—җ м ҖмһҘмқ„ н•ҳкІҢ лҗңлӢӨ.

м ҖмһҘлҗң Serving Data Storeмқҳ лҚ°мқҙн„°лҠ” кё°мҰқмқ„ н•ҙм„ң BatchлӮҳ Real-Time Processing Engineмқ„ нҶөн•ҙ м–ҙлҠҗ мҲҳмӨҖмқҳ лҚ°мқҙн„°лҘј м ҖмһҘн• м§Җ кІ°м •мқ„ н•ҳкІҢ лҗңлӢӨ.

[м ҖмһҘлҗң лҚ°мқҙн„°мқҳ мӮ¬мҡ©]

к·ёл ҮкІҢ м ҖмһҘлҗң лҚ°мқҙн„°лҘј кё°л°ҳмңјлЎң л¶„м„қмқҙлӮҳ DW(Data Warehouse)(м—¬лҹ¬ datasetл“Өмқ„ кҙҖл Ём„ұмһҲлҠ” кІғлҒјлҰ¬ л¬¶м–ҙм„ң м ҖмһҘн•ҙлҶ“мқҖ format)мқ„ кё°л°ҳмңјлЎң DM(Data Mart)лҘј кө¬м„ұн•ңлӢӨ.
DMлҠ” мӢңк°Ғнҷ” н•  мҲҳ мһҲлҠ” мһҘн‘ңлӢЁмң„лЎң кө¬м„ұмқ„ н•ҳкұ°лӮҳ, AIмҷҖ кҙҖл Ёлҗң Feature storeмқҳ к°ңл…җмңјлЎң AIлҘј лҸҢлҰҙ мҲҳ мһҲлҠ” datasetл“Өмқ„ кө¬м„ұн•  мҲҳ мһҲлӢӨ.

лҚ°мқҙн„° л¶„м„қк°Җл“Өмқҙ к·ёлһҳн”„лҘј нҶөн•ҙм„ң л¶„м„қмқ„ н•  мҲҳ мһҲкё° л•Ңл¬ём—җ, мөңмў…м ҒмңјлЎң лҚ°мқҙн„° мӢңк°Ғнҷ”лҘј н•ҳкІҢ лҗңлӢӨ.

[Raw Data Store -> AWS S3]

AWSм—җм„ңмқҳ Object storageмқё S3 Data Lakeм—җ м ҖмһҘмқ„ н•ҳкё° мң„н•ҙм„ң м•„лһҳмҷҖк°ҷмқҙ Batch-Processing Engineкіј Real-Time Processingм—җ mappingлҗҳлҠ” к°Ғ к°Ғмқҳ AWSмқҳ м„ңл№„мҠӨл“Өмқ„ м„ м •н•ңлӢӨ.

Amazon API Gateway / Amazon Kinesis Streams / Amazon kinesis Firehose / Amazon Pinpoint

мң„м—җ м •лҰ¬н•ң кө¬м„ұмңјлЎң м ‘м җмқ„ мһҳ м—°кІ°н•ҙм„ң нҢҢмқҙн”„лқјмқёмқ„ кө¬м¶•н•ңлӢӨ.

мң„мқҳ AWS м„ңл№„мҠӨл“Өмқ„ кө¬мІҙм ҒмңјлЎң м•Ңм•„ліҙмһҗ.

Amazon PinpointлҠ” мқјмў…мқҳ CRM м„ңл№„мҠӨлЎң, нҠ№м • мӣ№мқҙлӮҳ м•ұм—җм„ң л°ңмғқн•ҳлҠ” мқҙлІӨнҠёлҘј нҶөн•ҙ л§ҲмјҖнҢ…мқ„ н•  мҲҳ мһҲлҠ” нҲҙнӮ·мқҙлӢӨ.

мӢӨм ңлЎң Amazon API Gateway + Amazon Kinesis Streams кө¬м„ұмқ„ нҶөн•ҙм„ң лҚ°мқҙн„°лҘј мҲҳм§‘н•ҳлҠ” кІғмқҙ к°ҖмһҘ мўӢм§Җл§Ң, Amazon API Gatewayмқҳ л№„мҡ©м Ғ л¶ҖлӢҙмқҙ лҗ  мҲҳ мһҲлӢӨ.

л”°лқјм„ң Pinpoinмқҳ кІҪмҡ°м—җлҠ” SDKлҘј к°Җм§Җкі  мӣ№мқҙлӮҳ м•ұмқ„ л°°нҸ¬н•ҳкІҢ лҗҳл©ҙ, мҡ°м„  м ҖмһҘн•ҳлҠ”лҚ°к№Ңм§Җмқҳ л№„мҡ©мқҖ нҒ¬м§Җ м•Ҡкё° л•Ңл¬ём—җ, лӢӨмқҢкіј к°ҷмқҖ кө¬м„ұмңјлЎң Amazon Pinpoint -> Amazon Kinesis Stream -> Amazon Kinesis Firehose -> S3 лЎң кө¬м„ұн•ҳлҠ” кІғмқҙ мғҒлҢҖм Ғ л№„мҡ©м Ғ м Ҳк°җмқ„ к°Җм ёмҳ¬ мҲҳ мһҲлӢӨ.
мқҙмІҳлҹј л№„мҡ©м Ғ мёЎл©ҙмқ„ кі л Өн•ҳм—¬ мҲҳм§‘м—җ лҢҖн•ң лҳҗ лӢӨлҘё м•„нӮӨн…ҚмІҳлҘј л§Ңл“Ө мҲҳ мһҲлӢӨ.
(AWS мһҗкІ©мҰқмқ„ мқҙм ң л§ү кіөл¶Җн•ҳлҠ” мһ…мһҘмқҙм§Җл§Ң, мҷң л№„мҡ©м Ғ л¶Җл¶„м—җ лҢҖн•ң кіөл¶ҖлҸ„ н•„мҡ”н•ңм§Җм—җ лҢҖн•ҙм„ң мқҙм ң м•ҢкІҢ лҗҳм—ҲлӢӨ. м„ңл№„мҠӨ мҡҙмҳҒм—җ мһҲм–ҙ costмёЎл©ҙм—җм„ңмқҳ кі л ӨлҠ” мӨ‘мҡ”н•ҳлӢӨ.)

S3 Data Lakeм—җ лҚ°мқҙн„° м ҖмһҘлҗң мқҙнӣ„

S3м—җ лҚ°мқҙн„°к°Җ м ҖмһҘмқҙ лҗҳл©ҙ, н…ҚмҠӨнҠёлӮҳ JSON нҳ•нғңлЎң лҚ°мқҙн„°к°Җ м ҖмһҘмқҙ лҗңлӢӨ.
к·ё кё°л°ҳмңјлЎң лҚ°мқҙн„°м Ғ мҡ”кө¬мӮ¬н•ӯм—җ л”°лқјм„ң лҚ°мқҙн„°лҘј мІҳлҰ¬н•ҳкІҢ лҗҳлҠ”лҚ°, к°ҖмһҘ мқјл°ҳм ҒмңјлЎң Batch Processing Engineмқ„ нҶөн•ҙ лҚ°мқҙн„°лҘј мІҳлҰ¬н•ңлӢӨ.
нҶөмғҒм ҒмңјлЎң нҳ„мһ¬мӢңк°„-1мқј кё°мӨҖмқҳ лҚ°мқҙн„°лҘј к°Җм§Җкі  мІҳлҰ¬н•ңлӢӨ.
м ҖмһҘлҗң лҚ°мқҙн„°лҘј к°Җм§Җкі  мқҙм ң ETLкіјм •мқ„ м§„н–үн•ңлӢӨ. лҚ°мқҙн„°лҘј нҠёлһңмҠӨнҸјн•ҙм„ң м ҖмһҘмқ„ н•ҳлҠ”лҚ°, AWSм—җм„ң м ҖмһҘн• л•ҢлҠ” T1,T2мқҙлқјлҠ” мҡ©м–ҙлҘј мӮ¬мҡ©н•ңлӢӨ. (нҡҢмӮ¬л§ҲлӢӨ мӮ¬мҡ©лҗҳлҠ” мҡ©м–ҙ мғҒмқҙ)

м ҖмһҘлҗң лҚ°мқҙн„°лҘј DM лҚ°мқҙн„°лЎң м ҖмһҘн•ҳлҠ” мқҙ 3к°Җм§Җ мҠӨнғӯмқҙ ліҙнҶө мқјл°ҳм Ғмқё кіјм •мқҙлӢӨ.

S3 Data Lakeм—җ м ҖмһҘлҗң лҚ°мқҙн„° м „мІҳлҰ¬

S3 Data Lakeм—җ м ҖмһҘлҗң лҚ°мқҙн„°лҠ” мқјл Ёмқҳ лҚ°мқҙн„° м „мІҳлҰ¬ кіјм •мқ„ кұ°міҗм•јн•ңлӢӨ. м „мІҳлҰ¬лҘј мң„н•ҙм„ң мӮ¬мҡ©лҗҳлҠ” AWS м„ңл№„мҠӨм—җлҠ” Batch-ProcessingмӢңм—җ Spark/AWS DMSмқҙ н•„мҡ”н•ҳл©°, Real-time ProcessingмӢңм—җлҠ” Spark streaming/Amazon Kenesis Analyticsк°Җ мһҲлӢӨ.

м•һм„  нҸ¬мҠӨнҢ…м—җм„ңлҸ„ мқҙлҜё м •лҰ¬лҘј н–Ҳм§Җл§Ң, AWSм—җлҠ” EMRмқҙлқјлҠ” н•ҳл‘Ў Eco Systemкіј кҙҖл Ёлҗң м „л°ҳм Ғмқё м„ңл№„мҠӨк°Җ м„Өм№ҳлҗҳм–ҙмһҲлҠ” м„ңл№„мҠӨк°Җ мһҲлӢӨ. л°°нҸ¬н•ҳлҠ”лҚ° 10л¶„ лӮҙмҷёлЎң л§Өмҡ° м§§кё° л•Ңл¬ём—җ мҶҗмүҪкІҢ Batch-Processingмқ„ н•  мҲҳ мһҲлӢӨ.
AWSмқҳ DMS(Data Managing Search)лҠ” мқҙкё°мў… лҚ°мқҙн„°лҘј нҠёлһңмҠӨнҸјн•ҳлҠ”лҚ° мӮ¬мҡ©н•ңлӢӨ. мӣ№мқҙлӮҳ м•ұмқ„ OLTPн•ҳкё° мң„н•ҙ нҳ„мһ¬ м„ңл№„мҠӨм—җм„ң нҷңмҡ©лҗҳкі  мһҲлҠ” DBлҘј к°Җм ёмҷҖм•јлҗҳлҠ” кІҪмҡ°к°Җ мһҲлӢӨ. мҳҲлҘјл“Өм–ҙ, кі к°қ л°Ҹ мғҒн’Ҳ лҚ°мқҙн„°мқҳ л§ҲмҠӨн„°л¶„м„қмқ„ мң„н•ҙм„ң нҳ„мһ¬ мҡҙмҳҒмӨ‘мқё м„ңл№„мҠӨмқҳ лҚ°мқҙн„°лІ мқҙмҠӨм—җм„ң к·ёлҢҖлЎң лҚ°мқҙн„°лҘј к°Җм ёмҷҖм•јн•ңлӢӨ. мқҙ кІҪмҡ°м—җ DMSлҘј мӮ¬мҡ©н•ңлӢӨ.

SPARKм—җм„ң нҳ„мһ¬ мҡҙмҳҒмӨ‘мқё м„ңл№„мҠӨмқҳ л””л№„лҘј к°Җм ёмҳ¬ мҲҳлҸ„ мһҲлҠ”лҚ°, мқҙ кІҪмҡ°м—җлҠ” м„ңл№„мҠӨм—җ л¶Җн•ҳлҘј мӨ„ мҲҳ мһҲлӢӨ. мқҙлҹ¬н•ң мқҙмң л•Ңл¬ём—җ AWS DMSлҘј мӮ¬мҡ©н•ҳлҸ„лЎқ н•ңлӢӨ.

AWSмқҳ м„ңл№„мҠӨлҘј м„ нғқн• л•Ңм—җлҠ” м„ңлІ„л¶Җн•ҳмҷҖ мҪ”мҠӨнҠёл©ҙмқ„ кі л Өн•ҙм„ң м„ нғқн•ҙм•јн•ңлӢӨ.

Real-Time Processм„ нғқмӢңм—җлҸ„ AWSмқҳ EMRмқ„ мӮ¬мҡ©н•ҙм„ң к°„лӢЁн•ҳкІҢ Spark streamingк°Җ м„Өм№ҳлҗң нҷҳкІҪмқ„ кө¬м¶•н•ҙм„ң мӮ¬мҡ©н•  мҲҳ мһҲмңјл©°, Spark streamingмқ„ мӮ¬мҡ©н•ҳлҠ”лҚ° мһҲм–ҙ, лӢЁм җмқҖ н•ӯмғҒ мјңлҶ”м•јлҗҳлҠ” м җмқҙ мһҲкі , Amazon Kinesis AnalyticsлҘј нҶөн•ҙ мӢӨмӢңк°„мңјлЎң лҚ°мқҙн„°лҘј л¶„м„қн•  мҲҳ мһҲлӢӨ.

м „мІҳлҰ¬ мқҙнӣ„м—җ Serving Data StoreмҷҖ кҙҖл Ёлҗң AWS м„ңл№„мҠӨ

м „мІҳлҰ¬ мқҙнӣ„м—җ Serving Data StoreмҷҖ кҙҖл Ёлҗң AWS м„ңл№„мҠӨм—җлҠ” Amazon ES(Elastgic Search)/Amazon DynamoDB/Amazon RDSк°Җ мһҲлӢӨ.

Amazon ES(Elastic Search)

Amazon ESлҠ” лҚ°мқҙн„°лҹүмқҙ м§ҖмҶҚм ҒмңјлЎң л§Һм•„м§ҖлҠ” м„ңл№„мҠӨм—җлҠ” л¶Җм Ғн•©н•ҳлӢӨ. кі„мҶҚ м»ҙн“ЁнҢ… л°Ҹ мҠӨнҶ лҰ¬м§Җ мҳҒм—ӯмқ„ нҷ•мһҘн•ҙм•јлҗҳлҠ” н•„мҡ”м„ұмқҙ мһҲкё° л•Ңл¬ёмқҙлӢӨ. мқҙлЎңмқён•ҙ м§ҖмҶҚм Ғмқё мҡҙмҳҒмңјлЎңмқён•ҙ л№„мҡ©мқҙ л°ңмғқн•ңлӢӨ.
к·ёлҹј м–ҙл–Ө кІҪмҡ°м—җ Amazon ESк°Җ м Ғн•©н• к№Ң? мӢ к·ң м„ңл№„мҠӨлҘј лҹ°м№ӯн–Ҳмқ„л•Ң л“Өм–ҙмҳӨлҠ” м„ңл№„мҠӨ нғңк№…, к°Җмһ…, нҺҳмқҙм§•м—җ лҢҖн•ң л¶„м„қмқ„ л№ лҘҙкІҢ н•ҳкё° мң„н•ҙм„ң нҷңмҡ©н• л•Ң мң мҡ©н•ҳлӢӨ.
лҚ°мқҙн„° мң м§ҖлҠ” ліҙнҶө 2мЈјм •лҸ„н•ҳкі  м—¬мң к°Җ мһҲлӢӨл©ҙ н•ң лӢ¬м •лҸ„ мһЎкі  л¶„м„қн•ҳкё° мўӢмқҖ м„ңл№„мҠӨмқҙл©°,
Amazon DynamoDB, Amazon RDS(MySQLкіј к°ҷмқҖ DBMнҸ¬н•Ё)к°Җ мһҲлӢӨ.

Amazon Redshift

Amazonмқҳ RedshiftлҠ” м№јлҹј лІ мқҙмҠӨмқҳ л¶„м„қнҲҙлЎң, мҳЁн”„л ҲмқҙмҠӨ мҳӨлқјнҒҙмқҳ лҢҖн‘ңм Ғмқё м„ңл№„мҠӨл“Өмқ„ нҸ¬н•Ён•ңлӢӨ. лҚ°мқҙн„° л¶„м„қм—җ л§Һмқҙ мӮ¬мҡ©лҗңлӢӨ.

к·ёмҷём—җлҸ„ AWS лӮҙм—җ нҸ¬н•Ёлҗҳм–ҙмһҲлҠ” presto м„ңл№„мҠӨлҘј мӮ¬мҡ©н•ҙм„ң SQLмқ„ нҷңмҡ©н•ҙм„ң лҚ°мқҙн„°лІ мқҙмҠӨлҘј л¶„м„қн•  мҲҳ мһҲмңјл©°, м–ҙн”ҢлҰ¬мјҖмқҙм…ҳ мӨ‘м—җм„ң м ңмқј кҙңм°®м•„м„ң нҷңмҡ©лҸ„к°Җ лҶ’кі , мүҪкІҢ мӢңк°Ғнҷ”лҸ„ к°ҖлҠҘн•ҳлӢӨ.

мқҙлҹ° мқјл Ёмқҳ лҚ°мқҙн„° нҢҢмқҙн”„лқјмқём Ғ м ‘к·јмқ„ мқҙн•ҙн•ҳкі , лһҢлӢӨ м•„нӮӨн…ҚмІҳмқҳ кө¬м„ұмқ„ мқҙн•ҙн•ҳкі  мһҲлӢӨл©ҙ, лӮҳмӨ‘м—җ CSP(Cloud Service Provider)к°Җ л°”лҖҢм—Ҳмқ„ л•Ңм—җлҸ„ мқҙлҹ° м„ңл№„мҠӨл“Өмқ„ л§өн•‘н•ҳкі  лҚ°мқҙн„°мқҳ мқён’Ӣкіј м•„мӣғн’Ӣ нҸ¬л©§мқ„ м •мқҳн•ңлӢӨл©ҙ мүҪкІҢ нҢҢмқҙн”„лқјмқёмқ„ кө¬м¶•н•  мҲҳ мһҲлӢӨ.

ML/DLм—җм„ң мӮ¬мҡ©лҗҳлҠ” лҚ°мқҙн„° кө¬м„ұ

Serving Data Storeк°Җ кө¬м¶•мқҙ лҗҳкі , мһҗм—°мҠӨлҹҪкІҢ лҚ°мқҙн„°лҘј ліҙкІҢ лҗҳл©ҙ, Data Discovery/Predictive Modeling мҰү, ML/DLм—җ лҢҖн•ң лӢҲмҰҲ мҳҲмёЎк°’м—җ лҢҖн•ң лҚ°мқҙн„°лҘј кө¬м„ұн•  мҲҳ мһҲлӢӨ.
(мҡ°м„  м„ңл№ҷ лҚ°мқҙн„° мҠӨнҶ м–ҙлҘј м ңлҢҖлЎң кө¬м„ұн•ҳлҠ” кІғмқҙ мӨ‘мҡ”н•ҳлӢӨ!)
Serving DataлҘј м ңлҢҖлЎң кө¬м„ұн•ҙм•јл§Ң мҳҲмёЎк°’мқ„ кө¬н•ҳкё° мң„н•ң мҪ”мҠӨнҠёлҘј мӨ„мқҙкё° мң„н•ң л°©лІ•мӨ‘ н•ҳлӮҳмқҙлӢӨ.

Serving Data Storeк°Җ кө¬м„ұмқҙ м•Ҳлҗҳм—ҲлҠ”лҚ°, Raw data мҠӨнҶ м–ҙлҘј нҶөн•ҙм„ң Amazon SageMakerлӮҳ нҲҙл“Өмқ„ мӮ¬мҡ©н•ҙм„ң predict ML мҪ”л”©мқ„ мң„н•ҙм„ң л“ӨмқҙлҠ” мҪ”мҠӨнҠё ліҙлӢӨлҠ” лҚ°мқҙн„° м „мІҳлҰ¬м—җ лҢҖн•ң мҪ”мҠӨнҠёк°Җ нҒ¬кё° л•Ңл¬ём—җ мқҙ л°©лІ•мқҖ л№„м¶”н•ңлӢӨ.
(мҪ”мҠӨнҠёк°Җ л§Һмқҙл“ңлҠ” нҢҢмқҙн”„лқјмқё)

S3 Data Lakeм—җ мҢ“мқё лҚ°мқҙн„° кҙҖлҰ¬

S3м—җ лҚ°мқҙн„°к°Җ л§Һмқҙ мҢ“мқёлӢӨ. кіјкұ°м—җлҠ” DBMSлҘј л§Һмқҙ мӮ¬мҡ©н–Ҳкі , DBMSмқҳ лҚ°мқҙн„°лҘј кҙҖлҰ¬н•ҳкі  Archivingн•ҙм„ң н…Ңмһ… л“ңлқјмқҙлёҢм—җ м ҖмһҘмқ„ н–ҲлӢӨ.
нҳ„мһ¬лҠ” AWS м„ңл№„мҠӨ мӨ‘м—җм„ң Glacier м„ңл№„мҠӨк°Җ мһҲлҠ”лҚ°, S3мӨ‘м—җм„ң мөң н•ҳмң„ м ңмқј лҠҗлҰ° м„ңл№„мҠӨмқёлҚ°, Archiving н•ҳкё° мўӢлӢӨ. мқҙ м„ңл№„мҠӨм—җ Archiving н•  мҲҳ мһҲлҠ” мғқлӘ…мЈјкё°лҘј к°Җм ёк°ҖлҠ” кІғмқҙ мўӢкі , Data CatalogingмқҖ н•ҳл‘Ўк°ҷмқҖ кіім—җм„ң л©”нғҖмҠӨнҶ м–ҙлқјкі  н•ҳлҠ”лҚ°, л©”нғҖ мҠӨнҶ м–ҙлҘј нҶөн•ҙм„ң кҙҖлҰ¬н•ҙмӨҳм•ј л¶„м„қн•  мҲҳ мһҲлҠ” мӮ¬лһҢмқҙ л©”нғҖлҚ°мқҙн„°лҘј нҶөн•ҙм„ң л¶„м„қмқ„ л№ лҘҙкІҢ н•ҙмӨ„ мҲҳ мһҲлӢӨ.

S3 DataSecurity & Governance

мғқлӘ…мЈјкё°лҠ” Governanceм—җ н•ҙлӢ№лҗҳлҠ”лҚ°, мқҙ л¶Җл¶„мқ„ мһҳ кі л Өн•ҙм„ң нҢҢмқҙн”„лқјмқё кө¬м¶•мқ„ мӢңмһ‘н•ҙм•јн•ңлӢӨ. мІҳмқҢл¶Җн„° мҷ„лІҪн•  мҲҳ м—Ҷкі , л№„м–ҙмһҲлҠ” л¶Җл¶„мқҙ л§Һкі , л°ңм „мӢңнӮ¬ л¶Җл¶„м—җ лҢҖн•ҙм„ң кі л ӨлҘј н•ҙм•јн•ҳл©ҙм„ң мһ‘м—…мқ„ н•ҙм•јн•ңлӢӨ. мқҙмҷҖ кҙҖл Ёлҗң л§ҺмқҖ CSP м„ңл№„мҠӨл“Өмқҙ л“ұмһҘн•ҳкі  мһҲлӢӨ.

мқҙм ң м•һм„ң л°°мҡҙ нҢҢмқҙн”„лқјмқё кө¬м¶•м—җ м ‘к·јн•ҳлҠ” л°©лІ•мқ„ нҶөн•ҙ лӮҳл§Ңмқҳ нҢҢмқҙн”„лқјмқё, н”Ңлһ«нҸјмқ„ кө¬м„ұн•ҳлҸ„лЎқ л°ҳліөн•ҷмҠөмқ„ н•ҳкі  м—°мҠөмқ„ н•ҙм•јкІ лӢӨ.
